## 로그인_하기

- 유스케이스명
    - 로그인 하기
- 엑터
    - 관리자
- 시작 조건
    - 비로그인 상태
- 종료 조건
    - 관리자 권한으로 로그인이 되었다.
- 성공 시나리오
    - 사용자 : '관리자로 로그인하기' 버튼을 클릭한다. // 만들어야할듯 ;;
    - 시스템 : 관리자 로그인 페이지를 출력한다.
    - 사용자 : ID/PASSWORD를 입혁하고 '로그인' 버튼을 클릭한다.
    - 시스템 : 회원관리 페이지를 출력한다.    
- 예외 시나리오

- 유스케이스명
    - 회원 검색하기
- 엑터
    - 관리자
- 시작 조건
    - 관리자 권한으로 로그인 되어 있다.    
- 종료 조건
    - 회원관리 페이지에 해당하는 회원정보들이 출력된다.
- 성공 시나리오
    - 사용자 : 회원관리 페이지에서 검색정보를 설정하고 '검색' 버튼을 클릭한다.
    - 시스템 : 회원관리 페이지 목록에 검색정보에 해당하는 회원정보를 출력한다.

- 예외 시나리오
    - 사용자 : 해당하는 회원정보가 없다.
    - 시스템 : 회원관리 목록에 해당정보가 없다는 메시지를 출력한다.

- 유스케이스명
    - 메모 추가/수정하기
- 엑터
    - 관리자
- 시작 조건
    - 관리자 권한으로 로그인 되어 있다.
- 종료 조건
    - 회원의 메모 정보가 추가/수정된다.
- 성공 시나리오
    - 사용자 : 회원관리 페이지에서 해당하는 회원의 '메모수정' 버튼을 누른다.
    - 시스템 : 해당하는 회원의 메모란을 수정할 수 있게 text box형식(활성화)으로, 버튼명을 '수정확인'으로 전환해준다.
    - 사용자 : 수정할 text 입력 후 '수정확인' 버튼을 누른다.
    - 시스템 : 새로 입력한 메모 내용을 해당 회원 메모란에 출력해준다.

- 예외 시나리오
    - 사용자 : 메모할 수 있는 길이(용량)를 초과하였다.
    - 시스템 : 길이를 초과했다는 alert창을 출력한다.

- 유스케이스명
    - 회원 정지/해제하기
- 엑터
    - 관리자
- 시작 조건
    - 관리자 권한으로 로그인 되어 있다.
- 종료 조건
    - 회원의 등급을 정지 상태로 변경한다.
- 성공 시나리오
    - 사용자 : 회원관리 페이지에서 해당하는 회원의 '정지' 버튼을 누른다.
    - 시스템 : '유저ID 해당 회원을 정지시키겠습니까?'라는 alert를 출력해준다.
    - 사용자 : 해당 alert창에서 '확인' 버튼을 클릭한다.
    - 시스템 : 해당 회원의 상태를 '정지'로 기존 '정지' 버튼을 '정지해체'로 변경 출력한다.
    - (필요시) 메모 수정으로 해당 사유를 기록한다.

- 예외 시나리오

- 예외 시나리오

    - 시스템 : 등급관리 페이지 목록에 신청 정보가 없다는 메시지를 출력한다.
